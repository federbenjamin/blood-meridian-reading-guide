# Build notes

Developer notes for regenerating the bonus Kindle dictionaries. End users don't
need any of this — see the top-level [README](../README.md).

## Repository layout

```
README.md                                  user-facing guide
Blood_Meridian_Vocabulary_Companion.epub   the reading guide (the focal deliverable)
build_dicts.py                             generates the Webster-based dictionaries
dictionaries/                              the three bonus Kindle .mobi files
  Blood_Meridian_Dictionary_Extended.mobi  Webster 1913 + Blood Meridian terms folded in
  Webster_1913_Dictionary.mobi             Webster 1913 standalone
  Blood_Meridian_Dictionary.mobi           Blood Meridian terms only
docs/BUILD.md                              this file
src/Blood_Meridian_Dictionary_src/         OPF source of the standalone BM-only dictionary
tests/test_build_dicts.py                  unit tests for build_dicts.py
```

## What `build_dicts.py` does

It builds two of the three dictionaries — `Webster_1913_Dictionary.mobi` and
`Blood_Meridian_Dictionary_Extended.mobi` — from two inputs:

1. **Blood Meridian terms** — read directly out of
   `Blood_Meridian_Vocabulary_Companion.epub` (the `<p class="entry">` blocks in
   `OEBPS/epigraphs.xhtml`, `chapter-01..23.xhtml`, `epilogue.xhtml`). Multiple
   per-chapter senses of a word are merged into one multi-sense entry, each
   keeping its book quote.
2. **General definitions** — Webster's 1913 (public domain), a flat
   `{word: definition}` JSON from
   [matthewreagan/WebstersEnglishDictionary](https://github.com/matthewreagan/WebstersEnglishDictionary)
   (`dictionary.json`). The script downloads it to `webster_1913.json` on first
   run if it isn't already present.

In the Extended dictionary, a Blood Meridian word's entry shows its in-context
gloss + book quote first, then a `▸ Webster 1913` divider, then the general
Webster definition.

Run it:

```
python3 build_dicts.py            # writes the two .mobi files into dictionaries/
python3 build_dicts.py --help     # --webster / --companion / --kindlegen / --outdir
```

The third file, `Blood_Meridian_Dictionary.mobi` (Blood Meridian terms only), was
generated by an earlier script from `src/Blood_Meridian_Dictionary_src/` and is
not rebuilt by `build_dicts.py`.

## Webster entry formatting ("styled & spaced")

Raw Webster's 1913 definitions are one long run-on paragraph. `build_dicts.py`
reformats them for readability: one numbered sense per line, continuous numbering
1..N, italic domain labels like *(Astron.)*, small grey `[Obs.]` status tags,
dimmed illustrative quotations, and run-in sub-entries pulled into a **Phrases**
or **Synonyms** tail. This is verified lossless — no definition text is dropped
(see the tests).

## kindlegen

`kindlegen` is Amazon's legacy Mobipocket builder and the only thing that turns
the generated OPF source into a sideloadable dictionary `.mobi`. It ships inside
**Kindle Previewer** on macOS at:

```
/Applications/Kindle Previewer 3.app/Contents/lib/fc/bin/kindlegen
```

`build_dicts.py` finds it automatically (via `--kindlegen`, `$KINDLEGEN`, `PATH`,
or that bundled path). If it can't be found, the script writes the dictionary
source under `dictionaries/_build/` and prints the manual build command.

Gotchas:

- **Use `-c1` (PalmDOC).** `-c2` (Huffdic) segfaults on this content on macOS 15.
  `build_dicts.py` always passes `-c1`.
- **`dc:title` must be plain ASCII.** kindlegen does not decode numeric character
  entities in `<dc:title>`, so an em-dash or `&#8217;` shows up literally in the
  Kindle dictionary picker. Entities inside the entry bodies are fine.
- Kindle Previewer's own export only produces `.kpf` (KFX), which is *not*
  sideloadable as a device dictionary — only the bundled `kindlegen` makes a
  usable `.mobi`.

## Verifying a built `.mobi`

`epubcheck` does not apply (the dictionary uses proprietary `idx:` markup). To
sanity-check a build, confirm the binary contains the dictionary index records
(`INDX` / `TAGX` / `IDXT`), that the encryption flag is `0` (no DRM), and that the
`dc:title` reads correctly. PalmDOC-decompressing the text records and searching
for headwords confirms the content and styling landed.

## Tests

```
python3 -m unittest discover -s tests
```

Zero dependencies (stdlib `unittest`). The suite covers the pure formatting and
lookup-key logic, regression tests for bugs hit during development
(over-pluralization, the styling/attribute-quote collision, sense renumbering),
and a losslessness check, plus an integration test that parses the committed
companion EPUB.
